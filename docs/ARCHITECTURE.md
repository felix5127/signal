## 3. 架构决策记录 (ADR)

### ADR-001: 两阶段内容过滤

**状态**: 已实现 (后简化为统一过滤)

**背景**:
原始内容量大 (每日 500+)，直接 LLM 评估成本高。

**决策**:
采用两阶段过滤策略:
1. 规则预筛 (关键词、白名单) - 减少 70% 无效调用
2. LLM 评估 (0-5 分制)

**后续优化**:
合并为 `UnifiedFilter`，白名单直接通过，非白名单走 LLM。

---

### ADR-002: 三步分析法

**状态**: 已实现

**背景**:
单次 LLM 调用的分析质量不稳定。

**决策**:
采用三步分析:
1. **全文分析**: 提取摘要、观点、金句
2. **反思检查**: LLM 审视结果，提出改进
3. **优化整合**: 输出最终版本

**权衡**:
- 优点: 分析质量更高，减少幻觉
- 缺点: Token 消耗增加 ~2x

**可配置**: 支持跳过步骤 2/3 以节省成本。

---

### ADR-003: Resource 统一数据模型

**状态**: 已实现

**背景**:
早期有 Signal (v1) 和 Resource (v2) 两套模型。

**决策**:
统一使用 Resource 模型，支持 4 种内容类型:
- article: 文章
- podcast: 播客
- tweet: 推文
- video: 视频

**好处**:
- API 统一，前端逻辑简化
- 便于跨类型搜索和聚合

---

### ADR-004: pgvector 向量存储

**状态**: 已实现

**背景**:
研究工作台需要语义检索能力。

**备选方案**:
1. Pinecone (托管向量数据库)
2. Milvus (自托管)
3. pgvector (PostgreSQL 扩展)

**决策**: pgvector

**理由**:
- 无需额外基础设施
- 与现有 PostgreSQL 无缝集成
- 数据一致性有保障
- 性能满足当前规模 (<100k 向量)

**配置**:
- 维度: 512
- 索引: HNSW (m=16, ef_construction=64)
- 距离: cosine

---

### ADR-005: SSE 流式响应

**状态**: 已实现

**背景**:
研究任务和播客生成耗时较长 (30s-2min)。

**决策**:
使用 Server-Sent Events (SSE) 提供实时进度反馈。

**实现**:
```python
async def generate_stream():
    yield f"event: progress\ndata: {json.dumps(data)}\n\n"
```

**好处**:
- 用户体验好，实时反馈
- 比 WebSocket 更简单
- 原生浏览器支持

---

### ADR-006: 分形文档协议 (GEB)

**状态**: 已实现

**背景**:
代码和文档容易不同步，架构知识丢失。

**决策**:
采用三层分形文档结构:
- L1: 项目根目录 CLAUDE.md
- L2: 模块目录 CLAUDE.md
- L3: 文件头部注释

**强制回环**:
代码变更 → L3 更新 → L2 更新 → L1 更新

---

### ADR-007: 数据追踪 (飞书集成)

**状态**: 已实现

**背景**:
需要追踪流水线各节点的数据状态，便于分析和调优。

**决策**:
- 在 Pipeline 关键节点调用 DataTracker
- 批量写入飞书多维表格
- 失败不影响主流程

**追踪点**:
- 去重阶段: 记录重复项
- 过滤阶段: 记录 LLM 评分
- 存储阶段: 记录收录项

## 6. 扩展性设计

### 6.1 数据源扩展

新增数据源只需:

1. 创建采集器类 (继承 `BaseScraper`)
2. 实现 `scrape()` 方法
3. 在 `config.yaml` 添加配置
4. 注册到 Pipeline

```python
# backend/app/scrapers/new_source.py
class NewSourceScraper(BaseScraper):
    async def scrape(self) -> List[RawSignal]:
        # 实现采集逻辑
        pass
```

### 6.2 处理器扩展

新增处理器只需:

1. 创建处理器类
2. 定义输入/输出接口
3. 集成到 Pipeline

### 6.3 输出类型扩展

研究工作台支持新增输出类型:

1. 在 `ResearchOutput.output_type` 添加枚举值
2. 创建对应的 Agent/Generator
3. 添加 API 端点

### 6.4 水平扩展

**后端扩展**:
- 多实例部署 (Docker Swarm / K8s)
- Redis 作为共享缓存/队列
- 数据库读写分离

**前端扩展**:
- Cloudflare Pages 自动 CDN
- ISR (增量静态再生)
