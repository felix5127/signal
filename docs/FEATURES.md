## 1. 内容采集系统 (F1.x)

### F1.1 多源数据采集

**描述**: 从多个技术信息源自动采集原始内容。

**数据源清单**:

| ID | 数据源 | 采集器 | 频率 | 状态 |
|----|--------|--------|------|------|
| F1.1.1 | RSS/OPML 订阅 | `rss.py` | 每小时 | ✅ |
| F1.1.2 | Twitter/X (XGoing) | `xgoing.py` | 每小时 | ✅ |
| F1.1.3 | Podcast RSS | `podcast.py` | 按需 | ✅ |
| F1.1.4 | YouTube RSS | `video.py` | 按需 | ✅ |

**采集流程**:
```
1. 调用数据源 API/Feed
2. 解析响应，构造 RawSignal[]
3. 规则预筛 (关键词匹配)
4. URL Hash 去重
5. 返回新增信号列表
```

**验收标准**:
- [x] 每个数据源有独立的采集器实现
- [x] 采集器继承 `BaseScraper` 抽象类
- [x] 支持配置启用/禁用单个数据源
- [x] 采集失败不影响其他数据源
- [x] 记录每次采集的统计数据 (SourceRun)

**实现位置**: `backend/app/scrapers/`

### F1.3 三层去重机制

**描述**: 防止重复内容进入系统。

**去重策略**:

| ID | 策略 | 描述 | 阈值 |
|----|------|------|------|
| F1.3.1 | URL 精确匹配 | url_hash 完全相同 | 100% |
| F1.3.2 | 标题相似度 | Jaccard 相似度 | >0.8 |
| F1.3.3 | 内容指纹 | SimHash 指纹比对 | 可选 |

**验收标准**:
- [x] URL 相同的内容 100% 被去重
- [x] 标题高度相似的内容被检测
- [x] 去重日志可追溯

**实现位置**: `backend/app/processors/deduper.py`

## 2. 内容分析系统 (F2.x)

### F2.1 统一过滤器 (UnifiedFilter)

**描述**: AI 驱动的内容质量评估，决定是否收录。

**过滤流程**:
```
输入: title + content + url + source_name + source_is_whitelist
  ↓
Step 1: 白名单检查 → 白名单源直接通过 (score=5)
  ↓
Step 2: 语言检测 → 非中英文直接拒绝 (score=0)
  ↓
Step 3: 领域排除 → 命中排除词直接拒绝 (score=0)
  ↓
Step 4: LLM 评分 → 0-5 分 + 理由
  ↓
输出: FilterResult (score, reason, passed, is_whitelist, language, prompt_version)
```

**评分标准**:

| 分数 | 含义 | 处理 |
|------|------|------|
| 5 | 高价值，必须收录 | 通过 |
| 4 | 较高价值 | 通过 |
| 3 | 有一定价值 | 通过 (阈值) |
| 2 | 价值一般 | 拒绝 |
| 1 | 低价值 | 拒绝 |
| 0 | 无关/低质量 | 拒绝 |

**验收标准**:
- [x] 评分准确率 > 85% (与人工审核对比)
- [x] 白名单源自动通过
- [x] 非中英文内容自动过滤
- [x] 记录评分理由用于复查

**实现位置**: `backend/app/processors/unified_filter.py`

---

### F2.2 三步深度分析

**描述**: 对通过筛选的内容进行深度分析，生成结构化摘要。

**分析流程**:
```
Step 1: 全文分析
  → 摘要 (200-400字)
  → 核心观点 (3-5条)
  → 金句提取 (3-5条)
  → 标签分类
  → 评分 (0-100)

Step 2: 反思检查
  → LLM 审视 Step1 结果
  → 提出改进建议

Step 3: 优化改进
  → 整合反思意见
  → 输出最终版本
```

**输出字段**:

| 字段 | 类型 | 描述 |
|------|------|------|
| `one_sentence_summary` | string | 一句话摘要 |
| `summary` | string | 详细摘要 (200-400字) |
| `main_points` | array | 核心观点列表 |
| `key_quotes` | array | 关键引用 |
| `tags` | array | 标签分类 |
| `score` | int | 质量评分 (0-100) |
| `featured_reason` | string | 推荐理由 |

**验收标准**:
- [x] 摘要准确反映原文核心内容
- [x] 观点提取不遗漏关键信息
- [x] 三步分析可选择性跳过 (性能优化)

**实现位置**: `backend/app/processors/analyzer.py`

---

### F2.4 深度研究报告

**描述**: 为指定内容生成 1500 字深度研究报告。

**研究流程**:
```
1. 生成研究问题 (3-5 个)
2. Tavily 搜索每个问题
3. 整合搜索结果 + 原文内容
4. LLM 生成研究报告
```

**输出格式**:
```markdown
# {标题}

## 摘要
...

## 背景分析
...

## 核心发现
1. ...
2. ...

## 延伸思考
...

## 参考来源
...
```

**验收标准**:
- [x] 报告长度 1000-2000 字
- [x] 包含搜索增强的外部信息
- [x] 单篇成本控制 < $0.5
- [x] 支持策略选择 (LIGHTWEIGHT/AUTO)

**实现位置**: `backend/app/processors/deep_research/`, `backend/app/services/deep_research_service.py`
